Next Greater Element: Linear vs Circular

Key Difference

Linear Version

Input: [2, 1, 2, 4, 3, 1]

Rule: Find next greater element to the right only

Result: [4, 2, 4, -1, -1, -1]

Last elements often have no answer because there's nothing after them.

Circular Version

Input: [2, 1, 2, 4, 3, 1]

Rule: Array wraps around - after last element, continue from first

Result: [4, 2, 4, -1, 4, 2]

More elements get answers because we can "wrap around" to the beginning.

The Magic of "Processing Twice"

In circular version, we iterate 2 * n times:

  • First pass (i = 0 to n-1): Normal processing + push indices to stack
  • Second pass (i = n to 2n-1): Only pop from stack (no new pushes)

This simulates going around the circle once. Elements that couldn't find their next greater in the first pass get a second chance.
